How it's wired

The agent runs inside your network. Only aggregated metrics and query fingerprints leave the VPC — never rows, never literals.

   ┌──────────────────────┐           ┌──────────────────────┐
   │   Your VPC / net     │           │    VIGIL Cloud       │
   │                      │           │   (regional)         │
   │  ┌────────────────┐  │  mTLS     │                      │
   │  │ vigil-agent    │──┼──────────▶│  ingest → rollups    │
   │  │ (read-only DSN)│  │  outbound │  time-series + audit │
   │  └────────────────┘  │  only     │         │            │
   │          │           │           │         ▼            │
   │          ▼           │           │  alert engine, DSL,  │
   │   Postgres / MySQL / │           │  anomaly detectors   │
   │   MongoDB clusters   │           │         │            │
   │                      │           │         ▼            │
   │                      │           │  UI · API · copilot  │
   └──────────────────────┘           └──────────────────────┘

  • Query text is fingerprinted (hash of normalized SQL); literals are dropped.
  • Agent ↔ cloud: mTLS, workspace-scoped cert, rotatable via /governance API.
  • Outbound only — no inbound ports opened on your side.
  • EXPLAIN plans are captured on demand; agent refuses ANALYZE on DML.

What we store

Fingerprinted query IDs + aggregates (p50/p95/p99, counts, lock waits)

Schema metadata (tables, columns, types — no row data)

EXPLAIN plans you explicitly capture (literals redacted)

Immutable audit log of every admin action

What we don't

Row contents, PII, query parameters, or connection credentials

Anything from a schema you exclude from the agent config

FAQ

Does the agent need to see my data?

No. The agent reads pg_stat_statements / performance_schema / currentOp — aggregate counters and normalized query shapes, never row contents. You can additionally exclude schemas or databases in the agent config.

What happens if VIGIL Cloud is down?

Your database keeps running. The agent buffers metrics for up to 60 minutes and replays once the service returns. Alert rules fail-safe to not-fire, and your PagerDuty / Slack integrations are independent of VIGIL's UI.

Can I self-host and use Cloud at the same time?

Yes. The OSS edition and VIGIL Cloud use the same config format. Many teams self-host in staging and run Cloud in production, or vice-versa. You can migrate workspaces with a single vigil workspace export → import.

How does billing work?

Monthly or annual, per active user. Data ingest and retention are included within the plan's stated limits. See the pricing page for details.

Can I bring my own cloud account?

On the Enterprise plan, yes. We ship a Terraform module that provisions the control plane in your AWS / GCP / Azure account; we retain operator access only via short-lived tokens with a full audit trail.
