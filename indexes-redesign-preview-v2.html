What breaks if I apply this during peak hours?
✦
Short answer: nothing — CREATE INDEX CONCURRENTLY takes only a ShareUpdateExclusiveLock, so reads and writes continue. The build itself uses ~40 MB RAM and runs for ~3 min on your workload profile. It will block VACUUM FULL and CLUSTER on this table while it runs — neither of those is scheduled in the next 24h per pg_cron.
