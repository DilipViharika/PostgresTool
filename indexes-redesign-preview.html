↻ What's different vs. the current design

  • Distinct insights. Each AI item now shows a unique kind chip (MISS / UNUSED / BLOAT / DUP), a concrete impact (−ms, GB, scans/hr, hot-path context), a $-saved or GB-reclaimed delta, and a risk tag. No more five repeats of "95% confident".
  • Analysis is always visible. The right-hand panel is now a permanent rail — root cause, correlation, cost/benefit grid, risk block, safety checks and signals are there on first load. No need to click "Why?" to discover them.
  • Denser metric strip (8 tight cells, monospaced figures, micro-deltas) — the current design wastes ~40% vertical space.
  • Kind-coded colour system. Indigo = missing, slate = unused, pink = duplicate, orange = bloat. Severity is carried separately on the left rail.
  • Actionable rows. Inline checkbox for batch apply, Why + SQL + copy-suggestion actions per row, visual "applied" state with strikethrough + green tick.
  • Plan-as-a-file. The footer "plan.sql" button downloads a topologically-ordered, copy-pasteable fix script for the selected findings.
